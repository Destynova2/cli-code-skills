# Template — Tmuxinator Config (3-Tier Chain + per-role worktrees + safe wrapper)

Generate at `~/.config/tmuxinator/{session_name}.yml`.

## Embedded gotchas

- G1: `--permission-mode bypassPermissions` on the Chef (via wrapper)
- G4: `--teammate-mode tmux` (set conditionally inside the wrapper per role)
- G12: `on_project_first_start` does NOT exist, use `on_project_start`
- G24: The ccheck window is MANDATORY
- G32: Each worktree has its own `.claude/settings.local.json`. Agent spawns in the Chef prompt pass `cwd:`
- G33: The `apply` window only exists if the project has infra (tofu/terraform/helm/kubectl)
- G34: `--append-system-prompt "$(cat prompt.md)"` is **forbidden** — use `scripts/brigade-launch-agent.sh` instead (shell re-parsing breaks on backticks / `$(...)` / unbalanced quotes inside the markdown)
- G35: `git worktree add ... | head -1` under `pipefail` kills git via SIGPIPE — use `scripts/brigade-setup-worktrees.sh` instead
- G36: Each commis gets its own branch — the setup script handles this
- G37: First-time trust prompt appears even with `bypassPermissions` — ccheck or the user sends `Down Enter` on each claude pane after launch
- G38: Empty bash array + `set -u` explodes on bash 3.2 (macOS default) — the wrapper uses `${arr[@]+"${arr[@]}"}` conditional expansion

## Required artefacts (generated by the skill in Phase 2)

Before the tmuxinator YAML below can work, the skill must have generated:

1. `{project}/scripts/brigade-launch-agent.sh` (from `references/brigade-launch-agent.sh`, chmod +x)
2. `{project}/scripts/brigade-setup-worktrees.sh` (from `references/brigade-setup-worktrees.sh`, chmod +x)
3. `{project}/.claude/prompts/<role>-<session>.md` for each role that needs a claude pane (chef, ccheck, contre-chef-inter)
4. `{project}/.claude/prompts/<role>-<session>.initial.txt` for the same roles — holds the first user message (kept in a file so shell parsing does not mangle it)
5. `{project}/.claude/permissions-brigade/<role>/settings.local.json` for every role (see `permissions-template.md`)

---

```yaml
name: {session_name}
root: {project_path}

# --- Worktrees + per-role permissions (idempotent, G35) ---
# Delegates to scripts/brigade-setup-worktrees.sh which:
#   - creates one worktree per role on its OWN branch (G36)
#   - recovers from orphan state (dir exists but git doesn't know)
#   - copies the per-role settings.local.json from .claude/permissions-brigade/
#
# COMMIS_LIST = space-separated commis names (filled by the skill generator)
# BASE_BRANCH = integration branch each commis branch descends from
on_project_start:
  - COMMIS_LIST="{commis_names}" BASE_BRANCH="{base_branch}" {project_path}/scripts/brigade-setup-worktrees.sh {project_path} {session_name}

  # Pre-flight: every .claude/settings.local.json in place
  - |
    MISS=0
    for role in gate maitre{apply_role} vote-scope vote-secu vote-qualite {commis_role_names}; do
      test -f {project_path}-wt-${{role}}/.claude/settings.local.json \
        || {{ echo "MISSING: {project_path}-wt-${{role}}/.claude/settings.local.json"; MISS=1; }}
    done
    test $MISS -eq 0 || echo "=== ABORT: run 'make brigade-{session_name}-preflight' first ==="

on_project_stop:
  - |
    for role in gate maitre apply vote-scope vote-secu vote-qualite {commis_role_names}; do
      git -C {project_path} worktree remove {project_path}-wt-${{role}} --force 2>/dev/null || true
    done

windows:
  # --- CHEF: creates the team, spawns sous-chefs + commis + apply pane ---
  # The wrapper reads the prompt via $(< file) (G34) and sets --teammate-mode tmux (G4).
  - chef:
      root: {project_path}
      panes:
        - {project_path}/scripts/brigade-launch-agent.sh chef {session_name}

  # --- GATE: pure shell, Sous-Chef Merge sends commands here via tmux send-keys ---
  # CWD = gate worktree. Permissions: git push + gh pr (see permissions-template.md).
  - gate:
      root: {project_path}-wt-gate
      panes:
        - echo "=== GATE — merge, rebase, test, build, push, CI ===" && pwd

  # --- APPLY: infra mutations (tofu/helm/kubectl) after 3/3 quorum ---
  # Only present if project has infra tooling. CWD = apply worktree.
  # See references/apply-quorum.md.
  - apply:
      root: {project_path}-wt-apply
      panes:
        - echo "=== APPLY — tofu/helm/kubectl, gated by 3/3 quorum ===" && pwd

  # --- CCHECK: keyboard permission auto-approver (G24) ---
  - ccheck:
      root: {project_path}
      panes:
        - {project_path}/scripts/brigade-launch-agent.sh ccheck {session_name}

  # --- INTER: Agent Teams permission card forwarder (G29 — mandatory at tier ≥ M) ---
  - inter:
      root: {project_path}
      panes:
        - {project_path}/scripts/brigade-launch-agent.sh contre-chef-inter {session_name}
```

Notes:

- `{apply_role}` expands to ` apply` when the project has infra, empty string otherwise
- `{commis_role_names}` expands to `commis-bootstrap commis-scaleway ...`
- Omit the `apply` window block entirely when the project has no infra (G33)

## Architecture in tmux

```
Window 1: chef (cwd = {project}, wrapper script)
  ┌──────────────────┬─────────────┬──────────────┬──────────────┐
  │ Chef             │ Sous-Chef   │ Commis 1     │ Commis 2     │
  │ (plans, root     │ (gate       │ (wt-commis-1)│ (wt-commis-2)│
  │  settings)       │  worktree)  │              │              │
  └──────────────────┴─────────────┴──────────────┴──────────────┘

Window 2: gate (cwd = {project}-wt-gate, pure shell)
  ┌────────────────────────────────────────────────────────────────┐
  │ git push, gh pr merge, gh run watch                            │
  │ Denied: tofu/helm/kubectl apply, force-push                    │
  └────────────────────────────────────────────────────────────────┘

Window 3: apply (cwd = {project}-wt-apply, pure shell, only if infra)
  ┌────────────────────────────────────────────────────────────────┐
  │ tofu apply, helm upgrade, kubectl apply — after 3/3 APPROVE    │
  │ Denied: git push, git commit                                   │
  └────────────────────────────────────────────────────────────────┘

Window 4: ccheck (cwd = {project}, wrapper script)
  ┌────────────────────────────────────────────────────────────────┐
  │ Auto-approves keyboard permission prompts in normal zones      │
  └────────────────────────────────────────────────────────────────┘

Window 5: inter (cwd = {project}, wrapper script, tier ≥ M)
  ┌────────────────────────────────────────────────────────────────┐
  │ Agent Teams permission card forwarder                          │
  └────────────────────────────────────────────────────────────────┘
```

## First-time trust prompt (G37)

Even with `--dangerously-skip-permissions --permission-mode bypassPermissions`, Claude Code
shows a trust dialog the first time it sees a new working directory. The wrapper cannot
suppress this — the user (or ccheck) must send `Down Enter` to each claude pane after
launch. The skill generates a one-shot auto-accept snippet in `on_project_start`:

```yaml
  # After 3s, send Down+Enter to each claude pane (G37)
  - (sleep 3 && for w in chef ccheck inter; do tmux send-keys -t {session_name}:$w Down Enter; done) &
```

## Verification after generation

```bash
# Scripts executable
test -x {project_path}/scripts/brigade-launch-agent.sh && echo "launcher OK"
test -x {project_path}/scripts/brigade-setup-worktrees.sh && echo "setup OK"

# .initial.txt files present for claude-driven panes
for r in chef ccheck contre-chef-inter; do
  test -f {project_path}/.claude/prompts/${r}-{session_name}.initial.txt \
    || echo "MISSING initial: $r"
done

# Wrapper resolves prompt via $(< file) — no shell re-parsing
grep -q '\$(< ' {project_path}/scripts/brigade-launch-agent.sh && echo "safe load OK"
grep -q 'pipefail' {project_path}/scripts/brigade-setup-worktrees.sh && echo "DANGER: pipefail set" || echo "no pipefail OK"

# All per-role settings present
for role in gate maitre apply vote-scope vote-secu vote-qualite; do
  test -f {project_path}-wt-${role}/.claude/settings.local.json \
    && echo "perms $role OK" \
    || echo "perms $role MISSING"
done
```
