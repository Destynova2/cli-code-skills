# Template — Tmuxinator Config (3-Tier Chain + per-role worktrees)

Generate at `~/.config/tmuxinator/{session_name}.yml`.

## Embedded gotchas

- G1: `--permission-mode bypassPermissions` on the Chef
- G2: `--append-system-prompt "$(cat ...)"` not `--system-prompt-file`
- G4: `--teammate-mode tmux` to see Sous-Chefs + Commis
- G7: No `&` or background on claude
- G12: `on_project_first_start` does NOT exist, use `on_project_start`
- G13: `2>/dev/null || true` on git worktree add
- G24: The ccheck window is MANDATORY — without it, the Chef blocks on every worker permission
- G32: Each worktree has its own `.claude/settings.local.json`. The Agent spawn in the Chef prompt passes `cwd:` so Claude Code reads the right file. The tmux `root:` below is for the shell panes (gate, apply) that the Sous-Chef and apply pane drive via `tmux send-keys`.
- G33: The `apply` window only exists if the project has infra (tofu/terraform/helm/kubectl). Skip it otherwise.

---

```yaml
name: {session_name}
root: {project_path}

# --- Worktrees (idempotent — G13) ---
# One worktree per role. Each has its own .claude/settings.local.json.
on_project_start:
  # Role worktrees
  - cd {project_path} && git worktree add {project_path}-wt-gate         -b wt/gate         2>/dev/null || true
  - cd {project_path} && git worktree add {project_path}-wt-maitre       -b wt/maitre       2>/dev/null || true
  - cd {project_path} && git worktree add {project_path}-wt-apply        -b wt/apply        2>/dev/null || true
  - cd {project_path} && git worktree add {project_path}-wt-vote-scope   -b wt/vote-scope   2>/dev/null || true
  - cd {project_path} && git worktree add {project_path}-wt-vote-secu    -b wt/vote-secu    2>/dev/null || true
  - cd {project_path} && git worktree add {project_path}-wt-vote-qualite -b wt/vote-qualite 2>/dev/null || true
  # Commis worktrees (expanded from {worker_spawn_blocks})
{worktree_setup_commands}
  # e.g.: - cd {project_path} && git worktree add {project_path}-wt-commis-1 -b feat/feature 2>/dev/null || true

  # Permission files (generated by references/permissions-template.md)
  # These must exist BEFORE the Chef spawns the agents.
  - test -f {project_path}-wt-gate/.claude/settings.local.json         || echo "MISSING gate permissions"
  - test -f {project_path}-wt-maitre/.claude/settings.local.json       || echo "MISSING maitre permissions"
  - test -f {project_path}-wt-apply/.claude/settings.local.json        || echo "MISSING apply permissions"
  - test -f {project_path}-wt-vote-scope/.claude/settings.local.json   || echo "MISSING vote-scope permissions"
  - test -f {project_path}-wt-vote-secu/.claude/settings.local.json    || echo "MISSING vote-secu permissions"
  - test -f {project_path}-wt-vote-qualite/.claude/settings.local.json || echo "MISSING vote-qualite permissions"

on_project_stop:
  - cd {project_path} && git worktree remove {project_path}-wt-gate         2>/dev/null || true
  - cd {project_path} && git worktree remove {project_path}-wt-maitre       2>/dev/null || true
  - cd {project_path} && git worktree remove {project_path}-wt-apply        2>/dev/null || true
  - cd {project_path} && git worktree remove {project_path}-wt-vote-scope   2>/dev/null || true
  - cd {project_path} && git worktree remove {project_path}-wt-vote-secu    2>/dev/null || true
  - cd {project_path} && git worktree remove {project_path}-wt-vote-qualite 2>/dev/null || true
{worktree_cleanup_commands}

windows:
  # --- CHEF: creates the team, spawns sous-chefs + commis + apply pane ---
  # CWD = project root. The Chef reads the root .claude/settings.local.json
  # (which is read-only — the Chef never executes mutations directly).
  # GOTCHAS: G1 (bypassPermissions), G2 (append-system-prompt), G4 (teammate-mode tmux),
  # G31 (initial prompt as positional arg).
  - chef:
      root: {project_path}
      panes:
        - claude --dangerously-skip-permissions --permission-mode bypassPermissions --teammate-mode tmux --append-system-prompt "$(cat {project_path}/.claude/prompts/chef-{session_name}.md)" "Demarre IMMEDIATEMENT Phase 0 selon le system prompt : TeamCreate, recruter sous-chefs + maitre-d + apply-pane + commis, puis executer le PERT."

  # --- GATE: merge, CI, release (pure shell, no claude) ---
  # CWD = gate worktree. The Sous-Chef Merge sends commands here via
  # tmux send-keys. Permissions: git push + gh pr (see permissions-template.md).
  - gate:
      root: {project_path}-wt-gate
      panes:
        - echo "=== GATE — merge, rebase, test, build, push, CI ===" && pwd

  # --- APPLY: infra mutations (tofu/helm/kubectl) after quorum vote ---
  # CWD = apply worktree. Only present if project has infra tooling.
  # The apply pane agent drives this via tmux send-keys after 3/3 APPROVE.
  # See references/apply-quorum.md.
  - apply:
      root: {project_path}-wt-apply
      panes:
        - echo "=== APPLY — tofu/helm/kubectl, gated by 3/3 quorum ===" && pwd

  # --- CCHECK: permission auto-approver (the "contre-chef") ---
  # MANDATORY window. Runs a Claude instance that watches the Chef pane
  # and auto-approves permissions in normal zones, skips sensitive zones. (G24)
  - ccheck:
      root: {project_path}
      panes:
        - claude --dangerously-skip-permissions --permission-mode bypassPermissions --append-system-prompt "$(cat {project_path}/.claude/prompts/ccheck-{session_name}.md)" "Start watching chef pane and auto-approve permissions per the system prompt. Begin now."
```

## Architecture in tmux

```
Window 1: chef
  ┌──────────────────┬─────────────┬──────────────┬──────────────┐
  │ Chef             │ Sous-Chef   │ Commis 1     │ Commis 2     │
  │ (plans, root     │ (gate       │ (wt-commis-1)│ (wt-commis-2)│
  │  settings)       │  worktree)  │              │              │
  │ SendMessage →    │ push, PR,   │ commit only, │ commit only, │
  │ receives reports │ merge, CI   │ no push      │ no push      │
  └──────────────────┴─────────────┴──────────────┴──────────────┘

Window 2: gate (cwd = {project}-wt-gate)
  ┌────────────────────────────────────────────────────────────────┐
  │ Pure shell — git commands sent by the Sous-Chef                 │
  │ git push, gh pr merge, gh run watch                            │
  │ Denied: tofu/helm/kubectl apply, force-push                    │
  └────────────────────────────────────────────────────────────────┘

Window 3: apply (cwd = {project}-wt-apply, only if infra present)
  ┌────────────────────────────────────────────────────────────────┐
  │ Pure shell — apply commands after 3/3 quorum APPROVE           │
  │ tofu apply, helm upgrade, kubectl apply                        │
  │ Denied: git push, git commit                                   │
  └────────────────────────────────────────────────────────────────┘

Window 4: ccheck (cwd = {project})
  ┌────────────────────────────────────────────────────────────────┐
  │ Claude instance watching the Chef pane                          │
  │ Auto-approves normal zones, skips sensitive zones              │
  │ Logs every decision for traceability                           │
  └────────────────────────────────────────────────────────────────┘
```

## Notes

- **3-4 tmuxinator windows**: `chef`, `gate`, `apply` (optional), `ccheck`
- The Sous-Chefs, Commis, Maître, and Apply pane are spawned by the Chef via
  Agent Teams with explicit `cwd:` on each spawn — NOT via tmuxinator
- They appear in tmux panes thanks to `--teammate-mode tmux`
- `--permission-mode bypassPermissions` on the Chef bypasses UI prompts, but
  each agent still reads its own worktree's `.claude/settings.local.json`
  for its Bash allow/deny list (G32)
- The ccheck watches the Chef pane and handles keyboard permissions (G24)
- Maximum 5 commis + 1 sous-chef = 6 teammates + chef = 7 panes in window 1
